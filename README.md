Robot Spatial Data Processing Library

Библиотека для обработки пространственных данных на языке C. Проект реализует k-d дерево поиска и два алгоритма кластеризации (Fuzzy C-means, DBSCAN), которые используются в робототехнике для анализа лидарных карт и облаков точек.

Как запустить приложение (инструкция для пользователя)

1. Соберите программу

Откройте терминал в папке с проектом и выполните команду:

make

После этого появится файл robot_spatial (на Windows robot_spatial.exe).

2. Подготовьте файл с точками

Создайте CSV файл (например, data/points.csv) с координатами точек. Каждая строка содержит x,y:

Пример:
1.2,3.4
5.6,7.8
-0.5,2.0

3. Запустите нужную операцию

Откройте терминал в папке с проектом и выполните одну из команд:

Поиск ближайшей точки (к точке 1.0, 2.0):

На Windows:
.\robot_spatial.exe data/points.csv -kd_nearest 1.0,2.0

На Linux/Mac:
./robot_spatial data/points.csv -kd_nearest 1.0,2.0

Мягкая кластеризация на 3 кластера:

На Windows:
.\robot_spatial.exe data/points.csv -cmeans 3

На Linux/Mac:
./robot_spatial data/points.csv -cmeans 3

Кластеризация DBSCAN с радиусом 0.5 и минимальными 3 соседями:

На Windows:
.\robot_spatial.exe data/points.csv -dbscan 0.5,3

На Linux/Mac:
./robot_spatial data/points.csv -dbscan 0.5,3

4. Что означают параметры DBSCAN

- eps (0.5) – радиус поиска соседей.
- minPts (3) – минимальное количество соседей, чтобы образовать кластер.

Точки с пометкой -1 считаются шумом (не входят в кластеры).

5. Пример вывода для DBSCAN

Cluster labels:
Point 0: cluster 1
Point 1: cluster 1
Point 2: cluster -1
Point 3: cluster 2

Это означает:
- Точка 0 и точка 1 входят в кластер номер 1.
- Точка 2 не входит ни в один кластер (шум).
- Точка 3 входит в кластер номер 2.

6. Где посмотреть полный код и документацию

Весь код находится в папке src/. Основные файлы:
- main.c – обработка команд.
- kdtree.c – k-d дерево.
- fuzzy_cmeans.c – мягкая кластеризация.
- dbscan.c – DBSCAN.

Функциональность

- k-d tree – вставка точки, удаление точки, поиск ближайшего соседа.
- Fuzzy C-means – мягкая кластеризация точек с вычислением степеней принадлежности.
- DBSCAN – кластеризация на основе плотности с выделением шума.

Сборка проекта

1. Убедитесь, что установлен компилятор gcc и утилита make.
2. Откройте терминал в корневой папке проекта.
3. Выполните команду:

make

После успешной сборки появится исполняемый файл robot_spatial (на Windows – robot_spatial.exe).

Для очистки собранных файлов:

make clean

Общий формат команды:

robot_spatial <путь_к_csv_файлу> <операция> [параметры]

- <путь_к_csv_файлу> – файл с координатами точек (например, data/lidar.csv).
- <операция> – одна из трёх: -kd_nearest, -cmeans, -dbscan.
- [параметры] – дополнительные аргументы (зависят от операции).

1. Поиск ближайшей точки (k-d дерево)

Команда находит точку из вашего файла, которая находится ближе всего к заданной.

Пример для Windows:

.\robot_spatial.exe data\lidar.csv -kd_nearest 1.0,2.0

- data\lidar.csv – путь к файлу с точками.
- -kd_nearest – операция поиска ближайшего соседа.
- 1.0,2.0 – координаты точки, к которой ищем ближайшую.

Пример вывода:

Nearest point: 1.200000,3.400000

2. Fuzzy C-means (мягкая кластеризация)

Команда разбивает все точки на указанное количество кластеров и показывает центры этих кластеров.

Пример для Windows:

.\robot_spatial.exe data\lidar.csv -cmeans 3

- -cmeans – операция Fuzzy C-means.
- 3 – количество кластеров (можно указать 2, 4, 5 и так далее).

Пример вывода:

Centroids:
1.234567,2.345678
5.678901,7.890123
-0.123456,1.567890

Каждая строка – координаты центра одного кластера.

3. DBSCAN (плотностная кластеризация)

Команда группирует точки в кластеры на основе плотности, а точки, которые не попали ни в один кластер, помечает как шум.

Пример для Windows:

.\robot_spatial.exe data\lidar.csv -dbscan 0.5,3

- -dbscan – операция DBSCAN.
- 0.5 – радиус поиска соседей (eps). Если расстояние между точками меньше или равно 0.5, они считаются соседями.
- 3 – минимальное количество соседей (minPts). Точка считается плотной и образует кластер, если у неё есть хотя бы 3 соседа в радиусе 0.5.

Пример вывода:

Cluster labels:
Point 0: cluster 1
Point 1: cluster 1
Point 2: cluster -1
Point 3: cluster 2
Point 4: cluster 2
Point 5: cluster -1

- cluster 1, cluster 2 – номера кластеров (чем выше номер, тем позже найден кластер).
- cluster -1 – означает шум (точка не принадлежит ни одному кластеру).

Как интерпретировать вывод DBSCAN

- Точки с одинаковым номером кластера (например, cluster 1) образуют одну плотную группу.
- Точки с -1 – это выбросы или одиночные точки, которые находятся слишком далеко от плотных скоплений.

Формат входного CSV-файла

Файл должен быть в кодировке UTF-8 без заголовка. Каждая строка содержит координаты одной точки: x,y.

Пример содержимого data/lidar.csv:

1.2,3.4
5.6,7.8
-0.5,2.0
3.0,4.0
2.5,1.5
-1.0,-2.0
0.0,0.0
10.0,10.0


Структура проекта

robot_spatial/
├── Makefile
├── README.md
├── data/
│   └── lidar.csv
└── src/
    ├── main.c
    ├── utils.h
    ├── csv_reader.c
    ├── csv_reader.h
    ├── kdtree.c
    ├── kdtree.h
    ├── fuzzy_cmeans.c
    ├── fuzzy_cmeans.h
    ├── dbscan.c
    └── dbscan.h

- main.c – точка входа, обрабатывает аргументы командной строки.
- utils.h – определение структуры Point с координатами x и y.
- csv_reader.c и csv_reader.h – чтение точек из CSV-файла.
- kdtree.c и kdtree.h – реализация k-d дерева (вставка, удаление, поиск ближайшего соседа).
- fuzzy_cmeans.c и fuzzy_cmeans.h – алгоритм Fuzzy C-means (мягкая кластеризация).
- dbscan.c и dbscan.h – алгоритм DBSCAN (кластеризация по плотности).
